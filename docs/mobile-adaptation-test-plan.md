# 移动端兼容测试方案

## 目标

验证移动端兼容改造后：

- 不影响桌面端展示与核心交互
- 高优先级页面在手机上可用
- 本地开发支持模拟器和真机联调
- “明日之星 -> 单股诊断”链路满足 30 天 K 线规则

## 测试工具

### 1. 桌面模拟器

首选：

- Chrome DevTools Device Mode

可选补充：

- Responsively App

用途：

- 快速验证断点布局
- 检查抽屉、弹层、滚动和触控尺寸
- 同时预览多个视口

### 2. 自动化测试

主力工具：

- Playwright
- ExecuteAutomation Playwright

用途：

- `mobile/tablet/desktop` 多视口回归
- 核心链路自动化验证
- 截图回归和桌面端回归断言

### 3. 接口辅助验证

可选：

- Playwright API
- Postman / Insomnia

用途：

- 验证历史信息、Top 5、单股诊断等接口返回结构
- 为 UI 自动化准备稳定测试数据

### 4. 真机调试

至少覆盖：

- iPhone Safari
- Android Chrome

用途：

- 验证键盘弹出、地址栏视口变化、触摸滚动、图表 resize、安全区适配

## 本地联调准备

- [ ] 前端 dev server 支持 `0.0.0.0`
- [ ] 手机与电脑在同一局域网
- [ ] 手机可访问 `http://<LAN-IP>:5173` 或 `http://<LAN-IP>:8080`
- [ ] API 地址通过环境变量配置，不写死 `127.0.0.1`
- [ ] 验证登录、静态资源、接口请求、WebSocket 在手机端可用

### 推荐启动方式

完整链路验证：

```bash
./start.sh
```

手机访问：

- `http://<LAN-IP>:8080`

前端样式专项验证：

```bash
cd frontend
VITE_API_PROXY_TARGET=http://<LAN-IP>:8000 npm run dev:mobile
```

手机访问：

- `http://<LAN-IP>:5173`

验证顺序建议：

1. 先访问 `8080`，确认真实联调链路可用
2. 再访问 `5173`，用于快速调试样式和交互
3. 每次切换网络或重启服务后，重新确认手机端接口请求是否仍指向正确 LAN 地址

## 测试分层

## 第一层：桌面模拟器测试

推荐视口：

- `390x844`
- `430x932`
- `768x1024`
- `1024x768`
- `>=1280`

检查项：

- [ ] 无横向滚动
- [ ] 无文本溢出
- [ ] 无按钮遮挡
- [ ] 抽屉和弹层可用
- [ ] 图表尺寸正常
- [ ] 顶栏与底部布局稳定

重点页面：

- `TomorrowStar`
- `Diagnosis`
- `Watchlist`
- `Config`
- `Login`

## 第二层：Playwright 自动化测试

建议建立三套 viewport：

- `mobile`
- `tablet`
- `desktop`

自动化重点：

- [ ] 登录流程
- [ ] 移动端 `TomorrowStar` 仅展示历史信息和 Top 5
- [ ] 切换历史日期后 Top 5 正确刷新
- [ ] 点击 Top 5 跳转 `Diagnosis`
- [ ] 若来源为 `tomorrow-star`，移动端默认 30 天 K 线
- [ ] 桌面端 `Diagnosis` 多周期切换仍可用
- [ ] `Watchlist` 列表进入详情正常
- [ ] `Config` 表单可输入、可保存
- [ ] 桌面端原表格、双栏、菜单仍存在

截图回归建议：

- [ ] `TomorrowStar`
- [ ] `Diagnosis`
- [ ] `Watchlist`
- [ ] `Config`
- [ ] `Login`

## 第三层：真机测试

设备建议：

- 1 台 iPhone
- 1 台 Android

检查项：

- [ ] 页面首屏加载正常
- [ ] 抽屉导航可用
- [ ] 触控区域不难点
- [ ] 键盘弹出不遮挡主按钮
- [ ] 图表在旋转后正常 resize
- [ ] 长列表滚动顺畅
- [ ] 顶部状态条与安全区适配正常

## 第四层：桌面端回归测试

必须验证以下页面未退化：

- `TomorrowStar`
- `Diagnosis`
- `Watchlist`
- `Update`
- `Config`
- `Admin`

检查项：

- [ ] 原表格仍在
- [ ] 原双栏仍在
- [ ] 原图表切换仍可用
- [ ] 原导航和操作按钮不丢失
- [ ] 原功能链路可完成

## 关键业务链路

### 链路 A：移动端明日之星

- [ ] 打开移动端 `TomorrowStar`
- [ ] 查看历史信息列表
- [ ] 切换日期
- [ ] Top 5 正确刷新
- [ ] 点击 Top 5 股票进入单股诊断

### 链路 B：明日之星 -> 单股诊断

- [ ] 从移动端 `TomorrowStar` 进入
- [ ] 验证图表默认 30 天
- [ ] 验证桌面端不受影响

### 链路 C：重点观察

- [ ] 打开列表
- [ ] 进入详情
- [ ] 查看图表和建议
- [ ] 完成编辑或删除

### 链路 D：登录与配置

- [ ] 登录输入不被键盘遮挡
- [ ] 配置项输入不溢出
- [ ] 保存按钮始终可点击

## 可选质量补充

- [ ] 用 Lighthouse 做一轮移动端质量扫描
- [ ] 对高频页面补性能观测

## 测试输出要求

每轮测试至少记录：

- 测试日期
- 测试环境
- 设备或视口
- 覆盖页面
- 发现的问题
- 是否阻塞上线

## 通过标准

- 桌面端展示和主要交互不退化
- 移动端核心页面无横向滚动
- “明日之星”移动端仅展示历史信息和 Top 5
- 从移动端“明日之星”进入“单股诊断”时，K 线默认 30 天
- 至少完成一轮模拟器测试
- 至少完成一轮 iPhone 真机测试
- 至少完成一轮 Android 真机测试
- 至少完成一轮桌面端回归测试
