# 待办事项

## 验证清单

### 1. 环境准备

- [ ] 后端服务可正常启动，数据库连接为 PostgreSQL
- [ ] `intraday_analysis_snapshots` 表已创建
- [ ] 已配置有效的 `TUSHARE_TOKEN`
- [ ] 已准备 1 个管理员账号和 1 个普通用户账号
- [ ] 前一交易日已有候选股数据 `Candidate`
- [ ] 候选股对应股票已有足够历史行情 `StockDaily`

### 2. 管理员接口验证

- [ ] 管理员登录成功，能拿到 Bearer Token
- [ ] 调用 `GET /api/v1/analysis/intraday/status` 返回 200
- [ ] 在未生成快照前，`status` 为 `not_generated`，`has_data=false`
- [ ] 调用 `POST /api/v1/analysis/intraday/generate` 返回 200
- [ ] 生成成功后，返回 `status=ok`
- [ ] 生成成功后，`generated_count > 0`
- [ ] 再次调用 `GET /api/v1/analysis/intraday/status`，`has_data=true`
- [ ] 调用 `GET /api/v1/analysis/intraday/data` 返回 200
- [ ] 返回结果中包含 `trade_date`、`source_pick_date`、`snapshot_time`
- [ ] 返回结果中 `items` 不为空
- [ ] `items` 中字段包含 `code`、`name`、`b1_passed`、`score`、`verdict`、`signal_type`

### 3. 普通用户接口验证

- [ ] 普通用户登录成功，能拿到 Bearer Token
- [ ] 在 `12:00-15:00` 之外且没有快照时，调用 `GET /api/v1/analysis/intraday/status`
- [ ] 返回 `status=window_closed`
- [ ] 返回 `has_data=false`
- [ ] 在 `12:00-15:00` 之外调用 `GET /api/v1/analysis/intraday/data`
- [ ] 返回 200，但 `items=[]`
- [ ] 返回提示文案为“普通用户仅可在 12:00-15:00 查看中盘分析”或等价提示
- [ ] 在管理员已生成快照且处于 `12:00-15:00` 窗口时，普通用户调用 `GET /api/v1/analysis/intraday/data`
- [ ] 返回 200，且 `items` 可见

### 4. 权限验证

- [ ] 普通用户调用 `POST /api/v1/analysis/intraday/generate`
- [ ] 返回 403
- [ ] 非登录用户调用任一 `intraday` 接口
- [ ] 返回 401

### 5. 异常场景验证

- [ ] 前一交易日无候选股时调用生成接口
- [ ] 返回 `status=missing_source`
- [ ] Tushare 无数据或抓取失败时调用生成接口
- [ ] 返回 `status=fetch_failed`
- [ ] 候选股有实时行情但历史数据不足时
- [ ] 该股票被计入 `skipped_count`
- [ ] 重复调用生成接口时
- [ ] 同一 `trade_date + code` 不新增重复记录

### 6. 前端页面验证

- [ ] “明日之星” 页面存在 “中盘分析” 标签页
- [ ] 管理员可见 “手动生成” 按钮
- [ ] 普通用户不可见 “手动生成” 按钮
- [ ] 管理员生成后，页面能刷新出中盘分析结果
- [ ] 页面显示交易日、快照时间、来源日期
- [ ] 页面能显示 `B1`、`总分`、`信号类型`、`结论`、`趋势反转`
- [ ] 点击 “详情” 能跳转到单股诊断页
